# Guidelines

We will follow the below mentioned guidelines while creating Architecture Decision Records

## Prefixes

Each ADR will have a Prefix which helps us identify the type of architecture decision we are making. We will use following prefixes

|Prefix|Purpose|
|-----|-----|
|INFRA|Use this for ADRs associated with infrastructure provisioning|
|DOMAIN|Use this for any ADRs associated with domain decisions|
|DESIGN|Use this for any ADRs associated with design descisions|
|DEVPS|Use this for any ADRs associated with Devops decisions|

## Format

The ADR must contain following sections

|Section|Purpose|
|----|---|
|Title|Title of the decision record|
|Status|Status of the ADR. Can be Draft, Accepted, Rejected|
|Context|Context for which a decision is being made|
|Options Considered|Options which inflenced the decison, consider including detailed analysis here|
|Consequences|Consequences which will arise out of the decision, include both positive and negative where possible|
